# 环境配置

下载安装 golang（linux）

```
wget https://go.dev/dl/go1.19.3.linux-amd64.tar.gz
rm -rf /usr/local/go && tar -C /usr/local -xzf go1.19.3.linux-amd64.tar.gz
export PATH=$PATH:/usr/local/go/bin:/opt/go/bin
go version
```

配置环境变量

```
# 推荐配置
go env -w GO111MODULE=on(默认开启)
go env -w GOPROXY=https://goproxy.cn,direct

# /usr/local/go 为安装目录
go env -w GOROOT=/usr/local/go

# 可以放一些缓存/安装的二进制一类的（默认为$HOME/go）;/opt/go/bin 要配置到 PATH 里
go env -w GOPATH=/opt/go 
```

项目初始化

```
# 创建项目目录
mkdir project-name && cd project-name

# 如果是本地小项目/小工具
go mod init local/project-name

# 如果是业务项目,这样写的目的是为了标识你的项目和项目下的包，实现全局唯一性和可导入性。
go mod init github.com/yourname/project-name
go mod init gitlab.com/yourname/project-name
go mod init gitlab.com/group/project-name

```



项目目录结构

```
/project-name
├── cmd/
│   ├── app-a/
│   │   └── main.go        # 应用程序 A 的入口
│   └── app-b/
│       └── main.go        # 应用程序 B 的入口（多主程序模式）
│
├── internal/              # ⭐️ 内部包（最重要）
│   ├── app/               # 存放应用级别的业务逻辑，例如 service, handler, controller
│   └── platform/          # 存放通用的底层功能，例如 DB 封装, logger
│
├── pkg/                   # 外部共享包
│   └── util/              # 可被外部项目安全使用的通用函数库
│
├── api/                   # 外部接口定义
│   └── v1/
│       └── api.proto      # Protocol Buffers 或 OpenAPI/Swagger 定义
│
├── web/                   # Web 应用程序特定文件
│   ├── static/            # 静态资源（JS, CSS, 图片）
│   └── template/          # HTML 模板
│
├── vendor/                # 依赖副本（仅在使用 vendor 模式时存在）
├── scripts/               # 构建、安装、分析等脚本
├── configs/               # 配置文件模板或默认配置
├── deployments/           # 部署相关文件（Docker, K8s, Terraform）
├── test/                  # 额外的、大型的外部测试数据和工具
├── go.mod                 # 模块定义文件
├── go.sum                 # 模块依赖校验文件
└── README.md
```
