# 其他网络应用



## BGP Anycast

**BGP Anycast（边界网关协议任播）** 是一项非常精妙的技术，它让互联网从“一对一”的寻址方式变成了“就近服务”的模式。

简单来说，BGP Anycast 允许**多台地理位置不同的服务器使用同一个 IP 地址**



## GeoDNS (地理位置解析)

这是 Anycast 最常见的替代或互补方案。它的逻辑非常简单：

- **原理**：当用户请求域名解析时，DNS 服务器会根据用户的 **出口 IP 地址**（或通过 EDNS-Client-Subnet 协议获取的用户真实子网），查表判断用户所在的地理位置。
- **结果**：如果是日本用户，DNS 返回东京机房的 IP；如果是美国用户，则返回洛杉矶机房的 IP。



## GSLB (全局服务器负载均衡)

GSLB 是 GeoDNS 的“加强版”，它不仅看位置，还看**服务器的健康状况**。

- **智能决策**：它会实时监控全球各机房的负载、延迟和存活状态。
- **动态调度**：如果新加坡机房宕机了，GSLB 会立即感知，并在 DNS 层面把东南亚的用户导向香港或日本机房。
- **优势**：相比 Anycast 难以感知后端应用状态（BGP 只要路由器活着就行，不管 Web 服务死活），GSLB 对应用层更加敏感。
